  1. Pergunta 1: Qual é a diferença entre `==` e `===` em PHP?

    A principal diferença entre == e ===, é que utilizamos o operador == para verificar se os valores são iguais, realizando conversões de tipos quando necessário.

    Por outro lado, utilizamos o operador === para verificar se os valores e os tipos são estritamente iguais, sem realizar conversões de tipos.

    Operador Descrição
    == Verifica a igualdade de valor, convertendo os tipos se necessário.
    === Verifica a igualdade de valor e tipo (sem conversão de tipos).
    Clique sobre os operadores para obter um exemplo pratico
  2. Pergunta 2: Qual é a saída deste código em PHP?
    Código: 
    
                $x = 5;
                $y = 10;
                echo $x + $y . "10";

    A saída deste código é 1510 pois nesta expressão $x + $y é somado o valor "5 + 10" resultando em "15", em seguida o valor é concatenado a string "10" resultando no valor 1510. Clique no botão "Executar exemplo" para ver o resultado em um exemplo pratico.

  3. Pergunta 3: Como você incluir um arquivo em PHP?

    Existem duas maneiras que se pode incluir um arquivo PHP, utilizando include e require, utilizamos o require para que caso ele não encontre o arquivo que esta sendo incluído, ele emitirá um ERROR, fazendo a aplicação parar exatamente naquele ponto, já o require apenas emite um WARNING, seguindo executando o resto do script.

    Operador Descrição
    include inclui ou importar um arquivo, apresneta um WARNING se o arquivo especificado não for encontrado
    require inclui ou importar um arquivo, apresneta um ERROR se o arquivo especificado não for encontrado
    include_once inclui apenas uma vez um arquivo, apresneta um WARNING se o arquivo especificado não for encontrado
    require_once inclui apenas uma vez um arquivo, apresneta um ERROR se o arquivo especificado não for encontrado

    Além disso existem outras duas maneiras que se pode incluir um arquivo PHP, utilizando include_once e require_once, utilizando para evitará possíveis problemas como variáveis sobre escritas ou que funções e classes sejam redeclaradas causando um erro no código, visto que o arquivo será incluído apenas uma vez na execução do código.

  4. Pergunta 4: Qual a diferença entre `GET` e `POST` em relação ao envio de dados em uma solicitação HTTP?

    A diferença entre GET e POST, está na forma como os dados são enviados.

    Método Descrição Exemplo Prático
    GET Envia os dados como parte da URL. Visualizar detalhes do notebook
    POST Envia os dados no corpo da solicitação.

    Em uma requisição GET os dados são enviados via URL e geralmente são limitados a menos de 2.000 caractere, já uma requisição POST é enviado no corpo de requisição HTTP e não podem ser visto através da URL, além disso o método POST pode lidar com um grande volume de dados.

  6. Pergunta 6: O que é a função `unset()` em PHP e como ela é usada?

    A função unset() é utilizada para deletar uma variável, ou seja esta função remove qualquer valor que esteja assisado ao nome da variável, fazendo com que a variável não fique mais definida. a sintaxe desta função pode ser executada da seguinte maneira: 

    
                unset($variavel);

    Outro detalhe que é importante, é que quando uma variável é destruída com unset() ela não perde apenas o seu valor mais também a variável em si. Clique no botão "Executar exemplo" para ver o resultado em um exemplo pratico.

  1. Pergunta 7: Suponha que você tenha uma tabela chamada `produtos` com as colunas `id`, `nome`, `preco` e `data_criacao`. Escreva uma consulta SQL para selecionar o nome e o preço dos produtos com preço superior a 100 e ordenados pelo preço em ordem decrescente.

    Para fazer uma consulta SQL para selecionar o nome e o preço dos produtos com preço superior a 100 e ordenados pelo preço em ordem decrescente, devamos executar a seguinte consulta: 

    
                SELECT nome, preco
                FROM produtos
                WHERE preco > 100
                ORDER BY preco DESC;
  2. Pergunta 8: Você deseja criar uma classe PHP chamada `Logger` que possa ser usada para registrar mensagens de log em um arquivo. Como você implementaria essa classe?

    1. Declarar classe: Para implementar esta classe primeiro eu iria começar declarando a classe Logger 

    
                class Logger {

    2. Definir uma propriedade privada: Apos isso eu iria definir dentro da classe uma propriedade privada $logFile esta propriedade irá armazenar o nome do arquivo de log onde as mensagens serão salvas. 

    
                private $logFile;

    3. Criar o construtor da classe: o construtor da classe Logger é quem recebe o parâmetro $logFile que é o nome do arquivo de log onde as mensagens irão ficar armazenadas, o construtor irá atribuir esse valor a propriedade $logFile 

    
                public function __construct($logFile) {
                    $this->logFile = $logFile;
                }

    4. Criar um método publico: o método publico log da classe será usado para registra uma mensagem no arquivo de log, a variável $timestamp obtém a data e hora exata do registro, já o file_put_contents($this->logFile, $logEntry, FILE_APPEND) ira escrever a entrada do log no arquivo de log. 

    
                public function log($message) {
                    $timestamp = date("Y-m-d H:i:s");
                    $logEntry = "[$timestamp] $message\n";
                    file_put_contents($this->logFile, $logEntry, FILE_APPEND);
                }

    Apresentado do o código final desta forma:

    
                class Logger {
                    private $logFile;
            
                    public function __construct($logFile) {
                        $this->logFile = $logFile;
                    }
            
                    public function log($message) {
                        $timestamp = date("Y-m-d H:i:s");
                        $logEntry = "[$timestamp] $message\n";
                        file_put_contents($this->logFile, $logEntry, FILE_APPEND);
                    }
                }
  3. Pergunta 9: Qual é a diferença entre `INNER JOIN` e `LEFT JOIN` em SQL?

    A principal diferença entre INNER JOIN e LEFT JOIN é que o INNER JOIN retorna apenas os dados nas duas tabelas envolvidas na junção, ja o LEFT JOIN retorna os dados da tabela a esquerda(a primeira tabela mencionadna na consulta) e os dados da junção da tabela da direita(Segunda tabela mencionada na consulta)

  4. Pergunta 10: Qual é a finalidade da função `spl_autoload_register` em PHP?

    A função spl_autoload_register geralmente é usada para registrar funções de carregamento automático que são responsáveis por carregar automaticamente as classes. por exemplo, vamos utilizar a classe criada na resposta anterior

    Vamos supor que temos um arquivo chamado exemplo_autloder.php 

    

                    function autoloadMinhaClasse($nomeDaClasse) {
                        $caminhoDoArquivo = './php/' . strtolower($nomeDaClasse) . '.php';
                        if (is_file($caminhoDoArquivo)) {
                            include_once $caminhoDoArquivo;
                        }
                    }
                    spl_autoload_register('autoloadMinhaClasse');

    Podemos utilizar no index.php

    

                    require 'exemplo_autloder';

                    $logger = new Logger();
                    $logger->log();

    Podemos perceber que não foi declarada a classe Logger, então a função autoload vai tentar buscar o arquivo que está na pasta php/logger.php se o arquivo não for encontrado irá retornar um erro.

  5. Pergunta 11: Você está lidando com um grande conjunto de dados em PHP e precisa executar uma operação que pode levar muito tempo. Como você lidaria com isso para garantir que o script não exceda o tempo limite de execução do PHP?

    Para não exceder o tempo limite de execução do PHP existe diversas formas de lidar com isso, a primeira delas é aumentar o tempo limite de execução com a função set_time_limit, entretanto isso pode comprometer a performance do script, causando lentidões ou ate mesmo travando o servidor.

    Uma forma de não exceder o tempo limite de execução do PHP é dividir as tarefas em pequenas partes, permitindo que trabalhamos com um grande volume de dados de forma eficaz.
